## 1. 프로젝트 개요

### 1.1 배경 및 필요성

전력·에너지 솔루션 산업은 글로벌 에너지 전환, ESG 규제 강화, 재생에너지 확대 등 빠르게 변화하는 환경에 놓여 있다. 관련 뉴스와 정보는 국내외 수십 개의 미디어에 분산되어 있어, 구성원이 업계 동향을 효율적으로 파악하기 어렵다.

구성원은 각자 개별적으로 뉴스를 검색·수집하고 있으며, 영문 기사는 번역 과정에서 추가 시간이 소요된다. 이로 인해 중요한 시장 신호를 놓치거나 의사결정에 필요한 인사이트를 제때 얻지 못하는 경우가 발생한다.

### 1.2 프로젝트 목적

> **AI가 전력·에너지 산업의 국내외 뉴스를 자동으로 수집·번역·요약·분류하여,
> 전체 구성원이 언제 어디서나 한 곳에서 쉽게 열람할 수 있는 뉴스레터 대시보드를 구축한다.**

## 3. 목표 및 기대 효과

### 3.1 정량적 목표

| 목표 | 기준 |
|---|---|
| 업계 주요 뉴스 자동 수집 | 국내외 미디어 20개 이상, 일 평균 20~50건 |
| AI 요약 제공 | 전체 기사 99% 이상 한국어 요약 자동 생성 |
| 외국어 번역 자동화 | 영문 기사 제목·요약 한국어 번역 자동 처리 |
| 업데이트 주기 | 매일 오전 8시 자동 갱신, 격주 트렌드 리포트 격주 월요일 발행 |
| 접근성 | 로그인 없이 URL 하나로 전체 구성원 즉시 열람 |
| 기사 품질 | 에너지 주요 주제 기사만 수집 (AI 관련성 판별) |

### 3.2 정성적 기대 효과

- 구성원의 업계 정보 탐색 시간 단축
- 외국어 기사 접근 장벽 제거 → 글로벌 동향 파악 강화
- AI 트렌드 리포트를 통한 전략적 의사결정 지원
- 조직 전체의 업계 동향 공유 문화 형성

## 6. 시스템 아키텍처 개요

```
[수집 & AI 처리 파이프라인]              [서비스 제공]
GitHub Actions (매일 08:00 KST)  →    Next.js 정적 사이트 (Vercel)
  ↓                                      ↓
소스 목록 로드 (22개)               뉴스 카드 목록
  ↓                                 토픽 필터 + 키워드 검색
RSS 피드 파싱 / HTML 스크래핑        격주 트렌드 리포트
  ↓                                 날짜별 아카이브
다단계 품질 필터
  - URL 접근성 검사 (404 제외)      [데이터 저장]
  - 본문 10문장 이상 필터           JSON 파일 → GitHub 저장소
  - 에너지 관련성 필터 (키워드)     public/data/daily/YYYY-MM-DD.json
  ↓                                 public/data/biweekly/YYYY-BWnn.json
Gemini AI 처리                      public/data/index.json
  - 한국어 번역 (영문 기사)
  - 2~3문장 요약 생성
  - 토픽 분류
  - 에너지 주요 주제 판별 (isEnergyMain)
  ↓
중복 제거 → JSON 저장 → index.json 갱신
```

### 6.1 기술 선택 근거

| 선택 | 대안 대비 이점 |
|---|---|
| 정적 사이트(SSG) | 로그인 불필요, DB 없음, 운영 비용 최소 |
| Gemini API | 다국어 번역+요약 통합 처리, 비용 효율적 |
| 모델 체인 fallback | gemini-2.5-flash → gemini-3.1-flash-lite-preview → gemma-3-4b-it 순 자동 전환 |
| GitHub Actions | 무료 자동화, 코드 기반 이력 관리 |
| Vercel 배포 | Next.js 최적화, 자동 배포, 무료 플랜 |
