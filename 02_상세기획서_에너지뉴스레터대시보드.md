## 1. 서비스 구조 및 페이지 목록

### 1.1 사이트맵

```
/ (메인 — 오늘의 뉴스)
├── /archive (날짜별 아카이브)
│   └── /archive/[date] (특정 날짜 뉴스 목록)
├── /biweekly-report (격주 트렌드 리포트 최신호)
│   └── /biweekly-report/[reportId] (특정 격주 리포트)
├── /collect (수동 수집 관리 페이지)
└── /generate (리포트 수동 생성 페이지)
```

### 1.2 페이지별 역할

| 페이지 | URL | 역할 | 생성 방식 |
|---|---|---|---|
| 메인 | `/` | 최신 뉴스 카드 목록 + 필터 | SSG (매일 재빌드) |
| 아카이브 목록 | `/archive` | 날짜별 발행 이력 | SSG |
| 날짜별 뉴스 | `/archive/[date]` | 특정 날짜 뉴스 목록 | SSG |
| 격주 리포트 최신 | `/biweekly-report` | 최신 격주 트렌드 리포트 | SSG |
| 격주 리포트 특정호 | `/biweekly-report/[reportId]` | 특정 격주 리포트 | SSG |
| 수집 관리 | `/collect` | 크롤링 현황 및 수동 실행 UI | SSG |
| 리포트 생성 | `/generate` | 격주 리포트 수동 생성 UI | SSG |

## 2. 페이지별 상세 기능 명세

### 2.1 메인 페이지 (`/`)

#### 레이아웃 구조

```
┌────────────────────────────────────────────────────────────┐
│ [헤더]                                                      │
│  로고 + 서비스명 "에너지 인사이트"          [🔍 검색창]      │
│  최근 업데이트 시각 표시                                     │
├────────────────────────────────────────────────────────────┤
│ [토픽 필터 탭바]                                            │
│  [전체] [⚡전력인프라] [☀️에너지원] [🔧운영최적화]          │
│  [📋정책·규제] [🌿ESG·탄소중립] [📈시장동향]               │
├────────────────────────────────────────────────────────────┤
│ [뉴스 카드 그리드]                                          │
│  3열 (데스크톱) / 2열 (태블릿) / 1열 (모바일)               │
│  카드 × N개 (기본: 최신 30건 표시)                          │
├────────────────────────────────────────────────────────────┤
│ [더 보기] / 날짜별 피드 전환                                 │
├────────────────────────────────────────────────────────────┤
│ [푸터]                                                      │
│  업데이트 주기 안내 | 아카이브 링크 | 격주리포트 링크        │
└────────────────────────────────────────────────────────────┘
```

#### 기능 상세

| 기능 | 동작 방식 |
|---|---|
| 토픽 필터 | 탭 클릭 시 해당 토픽 뉴스만 표시 (클라이언트 필터링) |
| 검색 | 검색창 입력 시 제목·요약 텍스트 실시간 필터링 |
| 번역 배지 | 영문 원본 기사에 "번역" 배지 표시 (isTranslated = true) |
| 업데이트 시각 | 마지막 데이터 갱신 시각 헤더에 표시 |

```
┌──────────────────────────────────────────┐
│  [출처명]  2026-05-13  [번역] (영문시만)   │
│  ─────────────────────────────────────── │
│  국내 해상풍력 보급 목표 18GW로 상향        │
│  (원제: Korea raises offshore wind...)   │
│                                          │
│  정부가 2030년까지 해상풍력 설치 목표를     │
│  기존 12GW에서 18GW로 상향 조정했다.       │
│  재생에너지 확대 정책의 일환으로...         │
│                                          │
│  [☀️ 에너지원] [📋 정책·규제]             │
│                              [원문 보기→] │
└──────────────────────────────────────────┘
```

#### 카드 구성 요소 명세

| 요소 | 표시 내용 | 필수 | 비고 |
|---|---|---|---|
| 출처명 | 미디어 이름 (`source`) | 필수 | |
| 발행일 | YYYY-MM-DD 형식 | 필수 | |
| 제목 | 한국어 제목 (`title`) | 필수 | 영문 기사는 번역된 한국어 제목 |
| 원제 | 영문 원제 (`titleOriginal`) | 조건부 | 영문 기사만 표시 |
| [번역] 배지 | 외국어 원문인 경우 표시 | 조건부 | `isTranslated = true` 시 |
| AI 요약 | 2~3문장 한국어 요약 (`summary`) | 필수 | 최대 4줄 표시 |
| 토픽 태그 | 해당 토픽 배지 1~2개 | 필수 | |
| 원문 보기 | 원문 URL로 새 탭 이동 | 필수 | `target="_blank"` |

## 3. AI 파이프라인 상세 설계

### 3.1 전체 처리 흐름

```
Step 1: 소스별 수집
    - RSS 피드 파싱 (rss-parser 라이브러리)
    - HTML 스크래핑 (Cheerio)
    - PDF 크롤링 (IRENA 보고서)
    출력: 기사 후보 목록 {title, url, dateText}

Step 2: URL 접근성 검사
    - HTTP HEAD 요청으로 404/접속 불가 확인
    - 실패 시 수집 목록에서 완전 제외

Step 3: 에너지 관련성 필터 (키워드 기반)
    3-1. 제외 키워드 체크 (신뢰 소스 포함 전체 적용)
         예: 소설가, 코스피, 군수 후보, to lease, bundle deal
    3-2. 신뢰 소스 면제 (전기신문, 에너지경제신문, IEA, KEA 등)
         → 제외 통과 시 무조건 통과
    3-3. 혼합 소스 제목 키워드 매칭
         → 에너지 주제라면 제목에 반드시 키워드 등장

Step 4: 본문 길이 필터
    - 기사 본문 fetch (fetchBodyText)
    - 문장 수 10개 미만 기사 제외
    - 통과 시 bodyText 보존 (Step 6 fallback 및 Gemini content로 재활용)

Step 5: URL 중복 제거
    - 글로벌 중복: seen-urls.json 기준 이미 수집된 URL 제외
    - 배치 내 중복: 동일 실행 내 동일 URL 재등장 제거 (KEA 다중페이지 대응)

Step 6: Gemini AI 처리
    6-1. 번역 + 요약 + 토픽 분류 (통합 호출)
         - 한국어 기사: 요약(2~3문장) + 토픽 분류
         - 영어 기사: 한국어 제목 번역 + 한국어 요약 + 토픽 분류
    6-2. isEnergyMain 판별
         - true: 에너지가 핵심 주제 → 수집 진행
         - false: 에너지가 부수적 → 수집 제외 (seen에 등록, 재수집 방지)
    6-3. fallback (Gemini 실패 시)
         - summary: bodyText 앞 300자
         - title: 원제 그대로
         - topics: 키워드 기반 분류

Step 7: 데이터 저장
    - 일간 JSON: public/data/daily/YYYY-MM-DD.json
    - 인덱스 갱신: public/data/index.json
```

### 3.2 Gemini 모델 체인 및 레이트 리밋 처리

```
모델 우선순위:
  1. gemini-2.5-flash    (주력 — 429 즉시 다음 모델로)
  2. gemini-3.1-flash-lite-preview (2차 fallback — 503 시 8s 재시도 2회)
  3. gemma-3-4b-it       (최종 fallback)

요청 간격: 최소 10초 (GEMINI_INTERVAL_MS 환경변수로 조정)
429 처리: 재시도 없이 즉시 다음 모델 전환 (대기 시간 0)
503 처리: 8초 대기 후 재시도 (최대 2회)
```

### 3.3 Gemini 프롬프트 설계

#### 한국어 기사 (요약 + 토픽 + 관련성 판별)

```json
{
  "isEnergyMain": true,
  "titleKo": "기사 제목 (원문 그대로)",
  "titleOriginal": null,
  "summary": "2~3문장 핵심 요약. 구체적 수치나 정책명 포함.",
  "topics": ["전력 인프라", "정책·규제"]
}
```
- `isEnergyMain`: 에너지 산업·정책·기술·시장이 핵심 주제이면 `true`, 부수적이면 `false`
- `topics`: 6개 카테고리 중 최대 2개 선택

#### 영어 기사 (번역 + 요약 + 토픽 + 관련성 판별)

```json
{
  "isEnergyMain": true,
  "titleKo": "한국어로 번역된 제목",
  "titleOriginal": "원본 영어 제목",
  "summary": "2-3문장 한국어 요약. 구체적 수치와 기관명 포함.",
  "topics": ["에너지원"]
}
```

#### 격주 트렌드 리포트 프롬프트

지난 14일 기사 목록을 입력받아 다음을 생성:
- `headline`: 2주를 한 줄로 요약하는 헤드라인
- `topIssues`: 핵심 이슈 Top 5 (제목 + 2~3문장 설명)
- `topicSummaries`: 토픽별 동향 요약 (실제 기사 있는 토픽만)
- `keyPlayers`: 주목할 기업·기관 동향
- `nextWeekWatch`: 다음 주 주목 이슈 3개

### 3.4 에너지 관련성 필터 상세 (relevance-filter.js)

#### 제외 키워드 (신뢰 소스 포함 전체 적용)

| 언어 | 제외 키워드 |

`electimes`, `e2news`, `energydaily`, `todayenergy`, `energytimes`, `kea`, `igt`

> 참고: 전자신문(etnews)은 에너지와 무관한 기사(반도체, K-pop, 환율 등) 비중이 높아 신뢰 소스에서 제외됨

## 4. 데이터 모델

### 4.1 일간 뉴스 데이터 (`public/data/daily/YYYY-MM-DD.json`)

```json
{
  "date": "2026-05-13",
  "generatedAt": "2026-05-13T08:05:00+09:00",
  "articleCount": 25,
  "articles": [
    {
      "id": "a1b2c3d4-e5f6-7890-abcd-ef1234567890",
      "source": "PV Magazine",
      "sourceId": "pv-magazine",
      "sourceOrigin": "global",
      "originalLang": "en",
      "isTranslated": true,
      "title": "인도 석탄 회사, 태양광 제조 자회사 해산",
      "titleOriginal": "Coal India shuts down solar manufacturing arm",
      "summary": "인도 국영 석탄기업 Coal India가 태양광 제조 자회사 CIL Solar PV Ltd를 공식 해산했다. 이는 통합 태양광 PV 제조 사업 진출 계획을 공식적으로 철회한 것이다.",
      "topics": ["에너지원", "시장·가격 동향"],
      "publishedAt": "2026-05-12T09:00:00+09:00",
      "originalUrl": "https://www.pv-magazine.com/...",
      "collectedAt": "2026-05-13T08:02:14+09:00"
    }
  ]
}
```

#### 필드 설명

| 필드 | 타입 | 설명 |
|---|---|---|
| `id` | string | UUID v4, 기사 고유 식별자 |
| `source` | string | 미디어 표시명 |
| `sourceId` | string | 소스 식별 키 (sources.js의 id) |
| `sourceOrigin` | string | `"domestic"` / `"global"` |
| `originalLang` | string | `"ko"` / `"en"` |
| `isTranslated` | boolean | 영문 기사 번역 여부 |
| `title` | string | 표시 제목 (한국어; 영문 기사는 번역본) |
| `titleOriginal` | string\|null | 영문 원제 (영문 기사만) |
| `summary` | string | 2~3문장 한국어 요약 |
| `topics` | string[] | 토픽 태그 배열 (최대 2개) |
| `publishedAt` | ISO 8601 | 기사 발행 시각 |
| `originalUrl` | string | 원문 URL |
| `collectedAt` | ISO 8601 | 수집 시각 |

## 7. 자동화 파이프라인 명세

### 7.1 일간 수집 워크플로우 (`.github/workflows/daily-collect.yml`)

```yaml
name: Daily News Collection
on:
  schedule:
    - cron: '0 23 * * *'   # 매일 UTC 23:00 = KST 08:00
  workflow_dispatch:         # 수동 실행 가능

jobs:
  collect:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-node@v4
        with:
          node-version: '20'
      - name: Install dependencies
        run: npm ci
      - name: Run news collector
        env:
          GEMINI_API_KEY: ${{ secrets.GEMINI_API_KEY }}
        run: node scripts/run-crawl.js
      - name: Commit data
        run: |
          git config user.name "github-actions[bot]"
          git add public/data/ data/
          git diff --staged --quiet || git commit -m "chore: daily news update $(date +%Y-%m-%d)"
          git push
```

### 7.2 수집 스크립트 처리 순서 (`scripts/run-crawl.js`)

```
1. sources.js 로드 (크롤링 소스 목록)
2. RSS 크롤러 실행 (crawlRss) — rss/google-news 타입 소스
3. 스크래퍼 실행 (crawlScrape) — scrape 타입 소스
4. 결과 병합 및 날짜별 분류
5. 날짜별 JSON 저장 (배치 내 중복 제거 포함)
6. public/data/index.json 갱신
```

### 7.3 Gemini API 호출 최적화

| 전략 | 내용 |
|---|---|
| 요청 간격 | 최소 10초 (분당 6건, 레이트 리밋 여유 확보) |
| 429 처리 | 재시도 없이 즉시 다음 모델 전환 (대기 시간 0) |
| 503 처리 | 8초 대기 후 최대 2회 재시도 |
| 모델 체인 | gemini-2.5-flash → gemini-3.1-flash-lite-preview → gemma-3-4b-it |
| fallback | 모든 모델 실패 시 bodyText 앞 300자 사용 |

## 8. 유지보수 스크립트

### 8.1 요약 소급 보완 (`scripts/backfill-summaries.js`)

기존 수집 데이터의 빈 요약을 채우거나 영문 요약을 한국어로 번역하는 유지보수 스크립트.

```bash
# 빈 요약 전체 보완 (Gemini 사용)
node scripts/backfill-summaries.js

# 영어 기사 요약·제목 한국어 번역
node scripts/backfill-summaries.js --translate-en

# 특정 날짜 범위만
node scripts/backfill-summaries.js --start=2026-05-01 --end=2026-05-12

# 미리보기 (파일 저장 안 함)
node scripts/backfill-summaries.js --dry-run

# Gemini 없이 본문 앞 300자 fallback
node scripts/backfill-summaries.js --no-sum
```

#### `--translate-en` 모드 동작

1. `originalLang === 'en'` 이면서 `summary`에 한글이 없는 기사 대상
2. Gemini로 제목 한국어 번역 + 한국어 요약 생성
3. `title` = 한국어 번역 제목으로 업데이트
4. `titleOriginal` = 영어 원제 보존 (없으면 기존 title 저장)

### 8.2 중복 기사 제거 (`scripts/dedup-articles.js`)

```bash
# 전체 중복 제거
node scripts/dedup-articles.js

# 특정 날짜 범위
node scripts/dedup-articles.js --start=2026-05-01 --end=2026-05-12

# 미리보기
node scripts/dedup-articles.js --dry-run
```

### 8.3 비관련 기사 필터 (`scripts/filter-irrelevant-articles.js`)

이미 수집된 데이터에서 에너지 무관 기사를 소급 삭제.

```bash
node scripts/filter-irrelevant-articles.js
node scripts/filter-irrelevant-articles.js --dry-run
```

## 9. 에러 처리 및 예외 케이스

### 9.1 수집 단계 예외

| 예외 상황 | 처리 방법 |
|---|---|
| RSS 피드 접근 불가 | 해당 소스 스킵, 콘솔 경고, 다음 소스 진행 |
| 원문 링크 404 / 접속 불가 | 해당 기사 수집 목록에서 완전 제외 |
| 본문 10문장 미만 | 본문 짧음으로 수집 제외 |
| 에너지 무관 기사 (키워드) | 제목 키워드 미매칭 시 제외, 콘솔 로그 기록 |
| Gemini isEnergyMain: false | AI 판별 제외 + seen에 등록 (재수집 방지) |
| Gemini API 429 (레이트 리밋) | 즉시 다음 모델 전환 (대기 없음) |
| Gemini API 503 (서비스 불안정) | 8초 대기 후 최대 2회 재시도 |
| 모든 Gemini 모델 실패 | bodyText 앞 300자를 요약 fallback으로 저장 |
| JSON 파싱 오류 | 정규식으로 추출 시도 |
| 중복 기사 | URL 기준 스킵 (글로벌 + 배치 내) |

### 9.2 프론트엔드 예외

| 예외 상황 | 처리 방법 |
|---|---|
| 요약 없는 기사 | "요약 정보가 없습니다" 안내 텍스트 표시 |
| 데이터 파일 없는 날짜 | 아카이브에서 해당 날짜 비활성화 |
| 검색 결과 0건 | "검색 결과가 없습니다" 안내 텍스트 표시 |
