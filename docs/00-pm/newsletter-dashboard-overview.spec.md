> **문서 목적**: 서비스 전체 개요, 목표, 범위, 기대 효과를 이해관계자가 공유하기 위한 기획 요약 문서

## 1. 프로젝트 개요

### 1.1 서비스 소개

**에너지 인사이트 뉴스레터 대시보드**는 글로벌·국내 에너지 전문 소스 26개에서 기사를 자동 수집·분류·요약하고, 편집 담당자가 격주 뉴스레터를 분 단위로 제작할 수 있도록 지원하는 내부 웹 서비스입니다.

| 항목 | 내용 |
|------|------|
| **서비스명** | 에너지 인사이트 (Energy Insight) |
| **서비스 유형** | 사내 정적 웹 대시보드 (Vercel 배포, Next.js 14 SSG/CSR) |
| **주요 사용자** | 뉴스레터 편집 담당자 (1~2명), 구독 임직원 전체 |
| **운영 방식** | 매일 자동 크롤링 → 격주 뉴스레터 제작 |
| **현재 버전** | v2.4 (Sprint 7 완료) |

---

### 1.2 도입 배경

| 문제 | 원인 | 영향 |
|------|------|------|
| ① 소스 신뢰도 저하 | 기존 Google News RSS 집계 방식 → 지정 소스가 아닌 기사 수집 | 출처 추적 불가, 품질 불균일 |
| ② 뉴스레터 제작 시간 과다 | 수동 검토·선별·작성 (Notion·메일) | 편집 담당자 격주 수 시간 소요 |
| ③ 무관 기사 혼입 | 에너지 전문지에도 선거·부동산·연예 기사 포함 | 관련도 낮은 기사로 품질 저하 |
| ④ 기사 우선순위 없음 | 단순 수집 순서로만 나열 | 중요 기사 누락, 선별 효율 저하 |

---

### 1.3 해결 방안

```
[기존]  수동 검색 → 복사 붙여넣기 → 뉴스레터 작성 (수 시간)

[개선]  자동 수집 → AI 요약·분류 → 연관성 점수화 → 클릭 선택 → 즉시 생성 (10~20분)
```

## 2. 서비스 목표 및 KPI

### 2.1 핵심 목표

| # | 목표 | 측정 지표 |
|---|------|---------|
| 1 | 뉴스레터 제작 시간 단축 | 격주 작업 시간 ≤ 20분 (기존 대비 80% 감소) |
| 2 | 소스 신뢰도 확보 | 수집 기사 100% 지정 26개 소스 출처 |
| 3 | 수집 안정성 | 매 크롤링 시 26개 소스 중 20개+ 정상 수집 |
| 4 | 기사 품질 보장 | 에너지 관련성 필터 + 본문 10문장 이상 필터 통과 기사만 저장 |
| 5 | 편집 효율 향상 | 연관성 점수 상위 30개 자동 선별 → 담당자 최종 확인만 필요 |

- [x] `/collect`, `/screening`, `/generate` 3개 워크플로 페이지 정상 동작
- [x] 복사 버튼으로 뉴스레터 전문 1회 클릭 복사
- [x] 연관성 높음 기사가 상위에 표시 (스크리닝 점수 기준)
- [x] 확정 뉴스레터 아카이브 (`/newsletter-archive`) 조회 가능

## 3. 대상 사용자

### 3.1 주요 사용자

| 페르소나 | 역할 | 니즈 | 사용 빈도 |
|---------|------|------|---------|
| **뉴스레터 편집 담당자** | 격주 뉴스레터 제작 (1~2명) | 에너지 관련 기사를 빠르게 선별 → 뉴스레터 초안 확보 | 격주 1회 |
| **팀 열람자** | 구독 임직원 전체 | 카테고리별로 정리된 에너지 뉴스 소비 | 격주 1회 (수신) |

### 3.2 편집 담당자 사용 시나리오

```
1. 대시보드 접속 (/screening)
2. 연관성 점수 순 상위 30개 기사 확인
3. 불필요한 기사 체크 해제, 추가 기사 체크
4. [뉴스레터 생성하기] 클릭 (/generate)
5. 6개 카테고리로 자동 정렬된 뉴스레터 확인
6. [복사] 버튼 → 메일/메신저에 붙여넣기
```

## 4. 서비스 범위

### 4.1 In-Scope (포함)

| 영역 | 기능 |
|------|------|
| **크롤링** | 26개 지정 소스 RSS/스크래핑 직접 수집, 6단계 품질 필터 |
| **AI 처리** | Gemini API 번역·요약 (한국어 3문장), 6개 토픽 자동 분류 |
| **기사 열람** | 메인 페이지 기사 목록, 토픽 필터, 검색, 날짜별 아카이브 |
| **스크리닝** | 연관성 다차원 점수화, 카테고리 다양성 자동 선별 |
| **기사 선택** | 체크박스 선택, localStorage 저장, 전체 선택/해제 |
| **뉴스레터 생성** | 6카테고리 자동 분류, plain text 복사, 인쇄 |
| **아카이브** | 날짜별 과거 기사, 격주 리포트, 확정 뉴스레터 아카이브 |

### 4.2 Out-of-Scope (제외)

| 제외 항목 | 사유 |
|---------|------|
| 이메일 자동 발송 | 별도 메일 시스템 연동 필요 (향후 검토) |
| 기사 원문 저장 | 저작권 이슈 |
| 사용자 계정/권한 관리 | 사내 소규모 사용으로 불필요 |
| 실시간 알림 | 정적 서비스 구조상 서버 불필요 |
| 모바일 앱 | 웹 반응형으로 대응 |

## 5. 서비스 구조 (High-Level)

### 5.1 시스템 구성

```
┌─────────────────────────────────────────────────────┐
│  크롤링 파이프라인 (Node.js / Windows Task Scheduler) │
│                                                      │
│  26개 소스 → 수집 → AI 요약·분류 → JSON 저장          │
│  (매일 자동 실행)                                     │
└──────────────────┬──────────────────────────────────┘
                   │ public/data/*.json
                   ▼
┌─────────────────────────────────────────────────────┐
│  Next.js 14 웹 대시보드 (Vercel 정적 배포)            │
│                                                      │
│  [열람] 메인·아카이브·격주리포트 (SSG)                 │
│  [편집] 스크리닝·기사선택·뉴스레터생성 (CSR)           │
│  [관리] 뉴스레터 아카이브 (CSR)                       │
└─────────────────────────────────────────────────────┘
```

### 5.2 페이지 구성 요약

| 페이지 | 경로 | 목적 | 사용자 |
|--------|------|------|--------|
| 메인 | `/` | 최신 기사 열람, 토픽 필터, 검색 | 전체 |
| 기사 선택 | `/screening` | 연관성 점수 기반 기사 선별 + 선택 | 편집 담당자 |
| 기사 선택 (전체) | `/collect` | 전체 기사에서 직접 선택 | 편집 담당자 |
| 뉴스레터 생성 | `/generate` | 선택 기사 → 6카테고리 뉴스레터 | 편집 담당자 |
| 뉴스레터 아카이브 | `/newsletter-archive` | 확정 발송된 뉴스레터 조회 | 전체 |
| 날짜별 아카이브 | `/archive/[date]` | 특정 날짜 기사 열람 | 전체 |
| 격주 리포트 | `/biweekly-report/[id]` | AI 생성 트렌드 요약 리포트 | 전체 |

## 6. 핵심 기능 요약

### 6.1 수집 파이프라인 (6단계 품질 필터)

```
① 날짜 필터 (14일 이내)
② URL 중복 제거
③ 링크 유효성 검사 (404 제외)
④ 에너지 관련성 필터 (키워드 매칭)
⑤ 본문 길이 필터 (10문장 이상)
⑥ AI 판별 (isEnergyMain)
```

### 6.2 연관성 스크리닝 (14개 차원 점수화)

| 그룹 | 항목 | 최대 점수 |
|------|------|---------|
| 토픽 관련성 | AI 분류 토픽, 키워드 매칭, 광범위 에너지 | 무제한 |
| 전략 중요도 | 대형 규모·정책·산업구조·기술전환·시장긴급 | +30 |
| 사업 연관성 | SK 그룹, 소스 신뢰도(3단계), 국내/글로벌 영향 | +35 |
| 정보 품질 | 최신성, 요약 길이, 수량 패턴, 공식 기관 언급 | +18 |
| 중복 감점 | Jaccard 유사도 ≥ 0.4 | -15 |

**자동 다양성 선별**: 카테고리당 최대 8개, 소스당 최대 4개로 편중 방지

### 6.3 뉴스레터 자동 생성

- 선택 기사 → 6개 토픽 카테고리로 자동 분류
- 이모지 + 카테고리명 + 기사 제목·출처·요약·원문 URL 포함
- plain text 형식으로 1회 클릭 복사 (이메일·메신저 바로 붙여넣기)
- 인쇄 CSS 별도 적용 (A4 최적화)

## 9. 기대 효과

| 항목 | 기존 | 개선 후 | 효과 |
|------|------|---------|------|
| 뉴스레터 제작 시간 | 2~3시간 | 10~20분 | **85% 단축** |
| 소스 신뢰도 | Google News 집계 (불특정) | 26개 지정 소스 100% | **출처 투명성 확보** |
| 기사 선별 방식 | 담당자 전수 수동 검토 | AI 점수 순 자동 선별 → 담당자 최종 확인 | **선별 효율화** |
| 아카이브 관리 | 별도 보관 없음 | 날짜별·뉴스레터별 아카이브 자동 누적 | **이력 관리 가능** |
| 수요 기반 학습 | 없음 | 확정 뉴스레터 기반 수요 키워드 자동 반영 | **시간이 지날수록 정확도 향상** |

## 10. 제약 사항 및 리스크

| 리스크 | 발생 조건 | 대응 |
|--------|---------|------|
| CSS 셀렉터 변경 | 스크래핑 소스 사이트 리뉴얼 | warn 로그 모니터링 → 즉시 수정 |
| Cloudflare/SPA 차단 | 소스 사이트 보안 정책 강화 | RSS 대체 또는 Google News site: 필터로 우회 |
| Gemini API 사용 제한 | 기사 수 급증 시 | Rate limit 자동 대기 + fallback 키워드 분류 |
| localStorage 초기화 | 브라우저 데이터 삭제 | 선택 항목만 손실, 기사 데이터는 서버에 유지 |
| 수집 기사 0건 | 소스 전체 장애 | 날짜별 graceful 처리 (해당 날짜 skip) |
