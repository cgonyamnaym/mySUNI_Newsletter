## 1. 전체 화면 구조

### 1.1 레이아웃

모든 페이지는 **좌측 고정 사이드바 (너비 256px) + 우측 콘텐츠 영역** 구조를 따른다.

```
┌──────────────────────────────────────────────────────────────┐
│  Sidebar (고정, 256px)    │  Contents Area (가변)             │
│  ┌───────────────────┐   │  ┌──────────────────────────────┐ │
│  │  E 에너지 인사이트 │   │  │  Header (sticky, h-14)        │ │
│  ├───────────────────┤   │  ├──────────────────────────────┤ │
│  │  홈               │   │  │                               │ │
│  │  기사 선택  ← /screening  │  Page Content                │ │
│  │  뉴스레터 생성    │   │  │                               │ │
│  │  뉴스레터 아카이브│   │  │                               │ │
│  │  과거 아카이브    │   │  └──────────────────────────────┘ │
│  ├───────────────────┤   │                                   │
│  │  © 2026 Energy   │   │                                   │
│  └───────────────────┘   │                                   │
└──────────────────────────────────────────────────────────────┘
```

### 1.2 사이드바 네비게이션

| 메뉴명 | 링크 경로 | 비고 |
|--------|---------|------|
| 홈 | `/` | 최신 기사 메인 |
| 기사 선택 | `/screening` | 편집 담당자 주요 진입점 |
| 뉴스레터 생성 | `/generate` | 선택 완료 후 바로 접근 가능 |
| 뉴스레터 아카이브 | `/newsletter-archive` | 발송된 뉴스레터 이력 |
| 과거 아카이브 | `/archive` | 날짜별 기사 이력 |

> 현재 경로와 일치하는 메뉴는 파란색 활성 강조 처리됨.

## 2. 메인 페이지 (`/`)

### 2.1 페이지 목적

가장 최근 수집된 날짜의 기사를 토픽 필터·검색과 함께 열람하는 **기사 브라우징 페이지**.

### 2.2 화면 구성

```
┌─ Header (sticky) ─────────────────────────────────────────┐
│  E 에너지 인사이트  [뉴스레터 생성↗]  [격주 리포트↗]  업데이트: MM.DD HH:mm  [검색바] │
├───────────────────────────────────────────────────────────┤
│  [전체] [⚡전력 인프라] [☀️에너지원] [📋운영 최적화]        │
│  [📋정책·규제] [🌿ESG·탄소중립] [📈시장·가격 동향]          │
├───────────────────────────────────────────────────────────┤
│  기사 카드 그리드 (3열 → 2열 → 1열 반응형)                   │
│  ┌──────────┐ ┌──────────┐ ┌──────────┐                  │
│  │ NewsCard │ │ NewsCard │ │ NewsCard │                  │
│  └──────────┘ └──────────┘ └──────────┘                  │
├───────────────────────────────────────────────────────────┤
│  Footer: ⚡ 에너지 인사이트 · 뉴스레터 생성 → · 격주 리포트 →│
└───────────────────────────────────────────────────────────┘
```

### 2.3 기능 명세

| 기능 | 동작 | 비고 |
|------|------|------|
| 토픽 필터 | 칩 클릭 → URL `?topic=토픽명` 업데이트 → 해당 토픽 기사만 표시 | "전체" 선택 시 URL 파라미터 제거 |
| 검색 | 검색어 입력 → URL `?q=검색어` → Fuse.js 클라이언트 검색 | 제목·요약·소스명 대상 |
| 기사 카드 | 제목, 출처, 날짜, 토픽 뱃지, 번역 뱃지, 요약 (2줄), 원문 링크 | 원문 링크: 새 탭 열기 |
| 뉴스레터 생성 버튼 | Header 우측 파란 버튼 → `/collect` | 기사 선택 페이지로 이동 |
| 격주 리포트 | 최신 리포트 있을 때만 Header에 링크 표시 | |

### 2.4 데이터 소스

- **빌드 타임** `fs.readFileSync`로 최신 날짜 기사 로드 (SSG 정적 생성)
- URL 파라미터 처리는 클라이언트 사이드 (`useSearchParams`)

## 3. 기사 선택 · 스크리닝 (`/screening`)

### 3.1 페이지 목적

뉴스레터 편집 담당자가 **연관성 점수 기반으로 자동 선별된 상위 기사를 확인하고 선택**하는 주요 진입 페이지.

사이드바 "기사 선택" 메뉴의 기본 연결 경로.

### 3.2 화면 구성

```
┌─ Header ──────────────────────────────────────────────────┐
├───────────────────────────────────────────────────────────┤
│  ✨ 기사 선택                                              │
│                                                            │
│  [안내 배너]                                               │
│  최근 14일 기사 중 에너지 키워드 연관성이 높고 카테고리 다양성을 │
│  고려해 상위 30개를 자동 선별했습니다.                       │
│  수요 분석 활성 (아카이브 XX개 기사 기반) ●                  │
│                                                            │
│  표시 개수: [30개] [50개] [100개] [전체]  / 전체 XXX건       │
│                                                            │
│  [전체] [⚡전력 인프라] ... (토픽 필터)  뉴스 총 개수: XX    │
│  [현재 토픽 전체 선택 / 해제]                               │
│                                                            │
│  ┌──────────────────────────────────────────────────────┐ │
│  │ ☑ #1  [연관성 높음 85점]  |  에너지경제신문  |  05.20 │ │
│  │ [번역]  [⚡전력 인프라]                               │ │
│  │ 제목: 전력망 확충 2030년까지 100조 투자 확정...         │ │
│  │ 요약: ~~~~ (2줄)                                      │ │
│  │ 매칭: [전력망] [투자] [계통]                          │ │
│  │ 수요: [한전] [송전]                                   │ │
│  │ ⚡ 대형 규모  ⚡ 정책·규제                            │ │
│  │                                      [원문 보기 ↗]   │ │
│  └──────────────────────────────────────────────────────┘ │
│  ... (반복)                                                │
├───────────────────────────────────────────────────────────┤
│  [Sticky 하단 바]  XX개 기사 선택됨     [뉴스레터 생성하기 →]│
└───────────────────────────────────────────────────────────┘
```

### 3.3 자동 선별 로직 (Greedy Diversity Selection)

1. **점수 계산**: 기사별 연관성 점수 산출 (9절 상세 참조)
2. **중복 감점**: 상위 200개 대상 Jaccard 유사도 ≥ 0.4 → 후순위 -15점
3. **다양성 선별**:
   - 카테고리(primaryTopic)당 최대 8개
   - 동일 소스당 최대 4개
4. **기본 표시**: 상위 30개 (사용자가 50/100/전체로 변경 가능)

### 3.4 기능 명세

| 기능 | 동작 | 비고 |
|------|------|------|
| 표시 개수 선택 | 30/50/100/전체 버튼 클릭 → 목록 재산출 | 기본값: 30개 |
| 토픽 필터 | 칩 클릭 → 해당 토픽 기사만 표시 (screened 결과 내 필터링) | |
| 전체 선택/해제 | 현재 필터된 토픽 기준으로 일괄 선택 또는 해제 | |
| 기사 클릭 | 카드 전체 영역 클릭 → 선택/해제 toggle | 원문 링크 클릭은 별도 처리 |
| localStorage 저장 | 선택 변경 즉시 저장 | 새로고침 후에도 선택 유지 |
| 수요 분석 배지 | 아카이브 확정 기사 기반 자주 등장한 키워드 표시 (주황색) | |
| 전략 중요도 배지 | 5개 신호 탐지 시 보라색 ⚡ 배지 표시 | |
| SK 연관 배지 | SK 계열사 키워드 포함 시 빨간색 배지 표시 | |
| 중복 감점 배지 | Jaccard 유사도로 중복 판정된 기사에 회색 배지 | |
| 뉴스레터 생성 CTA | 선택된 기사 있을 때만 활성화 → `/generate` 이동 | 선택 0개: 비활성(회색) |

### 3.5 배지 색상 체계

| 배지 | 색상 | 의미 |
|------|------|------|
| 연관성 높음 (75점+) | 녹색 | 뉴스레터 적합도 높음 |
| 연관성 보통 (45~74점) | 주황색 | 참고 수준 |
| 연관성 낮음 (44점 이하) | 회색 | 포함 여부 검토 필요 |
| ⚡ 전략 중요도 신호 | 보라색 | 대형 사건·정책 등 주목 이슈 |
| SK 연관 | 빨간색 | SK 그룹 사업 직결 기사 |
| 중복 감점 | 회색 | 유사 기사 존재, 대표 기사 우선 선택 권장 |
| 수요 키워드 | 주황색 | 과거 뉴스레터에서 자주 다뤄진 토픽 |

## 4. 기사 전체 선택 (`/collect`)

### 4.1 페이지 목적

스크리닝 자동 선별 외에, **전체 수집 기사에서 직접 검색·필터하여 선택**하는 보조 페이지.

`/generate` 페이지의 "← 기사 선택으로" 버튼에서 진입.

### 4.2 화면 구성

`/screening`과 동일한 레이아웃. 주요 차이점:

| 항목 | `/screening` | `/collect` |
|------|-------------|-----------|
| 표시 개수 기본값 | 30개 | 50개 |
| 표시 개수 옵션 | 30/50/100/전체 | 50/100/전체 |
| 안내 문구 | "상위 30개 자동 선별" | "연관성 높은 순으로 표시" |
| Sidebar 연결 | "기사 선택" 메뉴 직접 연결 | 연결 없음 (generate ← 뒤로 가기로 접근) |

### 4.3 기능 명세

`/screening`과 동일. 단, 표시 개수 옵션이 **50/100/전체** (30개 없음).

## 5. 뉴스레터 생성 (`/generate`)

### 5.1 페이지 목적

선택된 기사를 **6개 토픽 카테고리로 자동 분류하여 뉴스레터 초안 형태**로 미리보기하고, 복사·인쇄하는 페이지.

### 5.2 화면 구성

```
┌─ Sticky 상단 바 ────────────────────────────────────────┐
│  [← 기사 선택으로]                    [복사]  [인쇄]     │
├──────────────────────────────────────────────────────────┤
│  (max-w-3xl 중앙 정렬)                                   │
│                                                          │
│  에너지 인사이트 뉴스레터                                  │
│  2026. 05. 15. ~ 2026. 05. 21.                          │
│                                                          │
│  ⚡ 전력 인프라                                           │
│  ─────────────────────────────────────────────          │
│  • [기사 제목] (출처)                                    │
│    요약 텍스트...                                         │
│    원문: https://...                                     │
│                                                          │
│  ☀️ 에너지원                                             │
│  ─────────────────────────────────────────────          │
│  • ...                                                   │
│                                                          │
│  (선택된 토픽 카테고리만 표시, 빈 카테고리 숨김)            │
└──────────────────────────────────────────────────────────┘
```

### 5.3 기능 명세

| 기능 | 동작 | 비고 |
|------|------|------|
| 기사 로드 | localStorage `newsletter-selection` → selectedIds 기준 필터링 | |
| 카테고리 분류 | TOPICS 정의 순서 기준 자동 그룹핑 | 미분류 기사는 별도 표시 |
| 빈 카테고리 처리 | 선택 기사 없는 카테고리는 섹션 자체 숨김 | |
| 기간 표시 | 선택 기사의 최소 publishedAt ~ 최대 publishedAt | |
| 복사 버튼 | plain text 형식으로 전체 뉴스레터 클립보드 복사 | 이메일·메신저 바로 붙여넣기 |
| 인쇄 버튼 | `window.print()` + `@media print` CSS 적용 (A4 최적화) | |
| 뒤로 가기 | `[← 기사 선택으로]` → `/collect` 이동 (localStorage 선택 유지) | |
| 선택 기사 없음 | "선택된 기사가 없습니다" 안내 메시지 + `/screening` 링크 | |

### 5.4 복사 텍스트 형식

## 6. 뉴스레터 아카이브 (`/newsletter-archive`)

과거에 확정·발송된 뉴스레터 목록을 조회하는 페이지.

### 6.2 데이터 구조

- `getArchiveEntries()` 함수가 localStorage 또는 별도 저장소에서 확정 뉴스레터 이력 로드
- 각 항목: 확정 날짜, 선택 기사 수, 포함 카테고리

### 6.3 기능 명세

| 기능 | 동작 |
|------|------|
| 목록 조회 | 확정 뉴스레터 역순 (최신 → 과거) 정렬 |
| 항목 클릭 | 해당 뉴스레터 상세 내용 표시 또는 `/generate` 재생성 |

## 7. 날짜별 아카이브 (`/archive`, `/archive/[date]`)

### 7.1 페이지 목적

특정 날짜에 수집된 기사를 열람하는 히스토리 조회 페이지.

### 7.2 `/archive` — 아카이브 목록

```
┌──────────────────────────────────────┐
│  과거 아카이브                         │
│                                       │
│  2026-05-21  (XX개 기사)              │
│  2026-05-20  (XX개 기사)              │
│  2026-05-19  (XX개 기사)              │
│  ...                                  │
└──────────────────────────────────────┘
```

- 빌드 타임 SSG. 수집된 날짜 목록 (`index.json.availableDates`) 전체 표시.

### 7.3 `/archive/[date]` — 날짜별 기사

- 해당 날짜 JSON 로드 → TopicFilter + NewsGrid 표시
- 이전/다음 날짜 네비게이션 버튼
- 메인으로 돌아가기 링크

| | 수량 패턴 | +5 | GW·MW·억원·% 포함 (regex 매칭) |

| | 소스 신뢰도 Tier1 | +10 | IEA, IRENA, BNEF |

| **전략 중요도** | 대형 규모 | +10 | 조원·GW급·다국적 투자 |
| | 정책·규제 | +10 | 법안 통과·고시·의무화 |

| | 기술 전환점 | +8 | 세계 최초·상용화 성공·기술 돌파 |
| | 시장 긴급 | +6 | 블랙아웃·전력 부족·가격 급등 |

### 9.2 Pass 2 — 중복 감점

- 상위 200개 기사 대상 pairwise 비교
- 제목 토큰 기반 **Jaccard 유사도** ≥ 0.4 → 후순위 기사 -15점 + `isDuplicate: true`
- 감점 후 재정렬

### 9.3 Pass 3 — Greedy Diversity Selection

| 단계 | 규칙 |
|------|------|
| 3-A (기본) | primaryTopic당 최대 8개 + 소스당 최대 4개 |
| 3-B (보완) | 개수 부족 시 primaryTopic cap 해제, 소스 cap만 유지 |
| 3-C (최종) | 그래도 부족 시 모든 cap 해제, 점수순 채움 |

### 9.4 수요 키워드 (아카이브 기반 학습)

- `getArchiveEntries()`로 과거 확정 뉴스레터 기사 전체 로드
- 제목·요약 토큰 빈도 분석 → 2회 이상 등장 토큰, 상위 50개 선정
- 현재 기사에 해당 토큰 포함 시 최대 +15점 가산
- **뉴스레터 확정 이력이 쌓일수록 정확도 향상** (자기 학습)

## 10. 크롤링 파이프라인 상세

### 10.1 수집 방식

| 타입 | 방식 | 소스 수 |
|------|------|---------|
| RSS | 공식 RSS 피드 직접 파싱 (rss-parser) | 17개 (15개 활성) |
| Scrape | Cheerio HTML 파싱 + CSS 셀렉터 | 8개 |
| PDF | PDF 직접 URL 수집 (IRENA) | 1개 |

### 10.2 6단계 품질 필터

```
[1] 날짜 필터
    publishedAt 기준 14일 이내인 기사만 통과
    ↓
[2] URL 중복 제거
    .crawled-urls.json 대조 → 이미 수집된 URL skip
    ↓
[3] 링크 유효성 검사
    HTTP 404 또는 접속 불가 → skip
    ↓
[4] 에너지 관련성 필터  ← isEnergyRelevant(title, summary, lang, sourceId)
    제목에 에너지 키워드 1개 이상 포함? → NO: skip (markSeen 미호출)
    신뢰 소스(TRUSTED_ENERGY_SOURCES)는 제외 키워드 통과 후 자동 pass
    ↓
[5] 본문 길이 필터  ← isBodyLongEnough(url)
    Cheerio로 본문 수집 → 문장 수 ≥ 10? → NO: skip
    timeout 8초 초과 / 페이월 / SPA 파싱 불가 → skip
    ↓
[6] AI 판별  ← summarize() via Gemini API
    isEnergyMain 플래그로 최종 확인
    번역(영→한), 요약(3문장), 토픽 분류(6개 카테고리)
    ↓
    markSeen() → daily JSON 저장
```

### 10.3 AI 요약·분류 (Gemini API)

| 항목 | 내용 |
|------|------|
| 모델 | gemini-2.5-flash |
| 입력 | 제목 + 본문 (최대 1500자) |
| 출력 | `{titleKo, summary(3문장), topics(최대 3개), isEnergyMain}` |
| Rate limit | 요청 간격 6,500ms 최소 유지 |
| 오류 대응 | 429 → Retry-After 대기 / 503 → 8초 fixed 대기 / fallback → 키워드 분류 |
| Fallback chain | gemini-2.5-flash → gemini-1.5-flash-latest → 키워드 기반 분류 |

### 10.4 에너지 관련성 필터 주요 예시

| 소스 | 제목 예시 | 판정 |
|------|----------|------|
| 에너지경제신문 | "BTS 귀환에 하이브 강세" | ❌ skip |
| 에너지경제신문 | "전기요금 인상 논의 본격화" | ✅ 통과 |
| 그린포스트코리아 | "게임 마케팅 크리에이터가 흥행 가른다" | ❌ skip |
| Electrek | "BYD drop-top electric hypercar images" | ❌ skip (제외 키워드: bundle deal류) |
| Electrek | "EV charging infrastructure expands" | ✅ 통과 |
| Carbon Brief | 에너지 관련 없어 보여도 | ✅ TRUSTED_ENERGY_SOURCES 면제 |

| 링크 유효성 실패 (404) | isUrlAccessible → false → skip | — |

### 12.2 CSR 페이지 (`/screening`, `/collect`, `/generate`)

| 상황 | 처리 | UI 표시 |
|------|------|--------|
| 데이터 로딩 중 | 로딩 스피너 | "연관성 분석 중입니다..." |
| localStorage 없음/만료 | 빈 선택으로 초기화 | 기사 선택 안내 |
| daily JSON fetch 실패 | 해당 날짜 skip, 나머지 날짜 표시 | 자동 처리 (사용자 인지 불필요) |

| 복사 API 미지원 브라우저 | — | 별도 처리 미적용 (최신 브라우저 대상) |
